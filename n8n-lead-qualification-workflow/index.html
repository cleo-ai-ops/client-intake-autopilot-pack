Fast answer: use a form/webhook trigger, normalize the answers, calculate a 0–12 fit score, split leads into hot/warm/cold paths, then send hot leads your booking link and send warm leads one clarifying question.

The simplest n8n workflow map

✓ 1. Trigger

Use a Webhook node, n8n Form Trigger, Tally webhook, Typeform, or Google Forms via Sheets trigger.

✓ 2. Normalize

Map messy form fields into consistent variables: budget, deadline, service, authority, scope, and readiness.

✓ 3. Score

Use a Set or Code node to assign points. Deterministic scoring is easier to trust than an AI black box.

✓ 4. Route

Use Switch or IF nodes to send hot, warm, cold, and scope-risk leads down different paths.

Copy this routing logic into n8n

if score >= 9 and scope_risk is false: tier = "hot" action = "send booking link + personal reply" elif score >= 6: tier = "warm" action = "ask one clarifying question" elif scope_risk is true: tier = "risk" action = "send boundary-setting reply before any call" else: tier = "cold" action = "send polite decline or resource"

In n8n, this can be a Code node followed by a Switch node with four branches.

Recommended node sequence

  1. Webhook/Form Trigger: receive the client intake submission.
  2. Set: rename fields into stable names like budget_range, deadline, and decision_maker.
  3. Code: calculate budget_score, timeline_score, authority_score, service_fit_score, scope_score, readiness_score, total_score, and tier.
  4. Google Sheets: append the original answers, score, tier, and next action.
  5. Switch: branch hot, warm, cold, and scope-risk leads.
  6. Gmail/SMTP: send the matching reply template.
  7. Optional notification: send yourself a Telegram/Slack/email alert only for hot leads.
